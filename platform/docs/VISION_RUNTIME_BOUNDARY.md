# 视觉层与工序判定层边界

## 1. 基本原则

视觉算法负责提供观察证据，SOP Runtime 负责业务判定。

任何 YOLO 标签、跟踪框、手部关键点或动作分类结果都不能直接把工序设置为 `PASS`。这样可以避免更换模型时破坏流程，也能防止单帧误检直接形成错误生产记录。

## 2. 视觉处理链

```text
相机帧
  -> 目标检测
  -> 多目标跟踪
  -> 手部/工具关联
  -> ROI 与空间关系
  -> 状态稳定和置信度评估
  -> Observation
  -> SOP Runtime
  -> Judgement / Deviation / Evidence
```

首个包装场景推荐使用：

- YOLO：纸盒、耳机、充电器、绿色小袋等目标检测。
- ByteTrack 或等价跟踪器：维护物体身份和移动轨迹。
- MediaPipe Hands 或轻量手部模型：提供手与物体交互证据。
- 几何关系：判断盒外、盒内、进入、离开、包含和接触。
- 状态稳定器：过滤单帧抖动和短时遮挡。
- 通用 Runtime：处理前置条件、状态转移、保持时间、顺序和异常。

复杂装配动作在规则方案无法达到指标时，才增加 TCN、LSTM 或视频 Transformer，并仍然通过统一 `Observation` 或语义事件契约接入。

## 3. Observation 责任

视觉适配器输出的观察至少包括：

- 单调递增时间戳。
- 规范化事实，例如 `box.state=open`。
- 交互证据，例如 `hand:earphone`。
- 数据源是否健康。
- 当前证据是否足以可靠判断。
- 对应帧或视频缓存引用。

视觉适配器必须处理：

- 检测标签到领域事实的映射。
- 目标 ID 和轨迹连续性。
- 短时遮挡。
- 坐标系和 ROI。
- 置信度与画面质量。

视觉适配器不得处理：

- 当前 SOP 执行到哪一步。
- 某步骤是否通过。
- 错序、漏做或超时结论。
- 页面状态或数据库业务状态。

## 4. 纸盒状态的特殊处理

物品放入盒内后，YOLO 可能不再持续输出 `box_open`。因此适配器不能简单把“当前帧没检测到 box_open”解释成盒子已关闭。

正确策略：

1. 通过手部参与的 `closed -> open` 可靠转移确认开盒。
2. 在没有可靠关盒证据前，维持逻辑上的“已打开”状态。
3. 允许盒体短时丢检或被物品遮挡。
4. 只有手部参与且观察到可靠的 `open -> closed` 转移，才形成关盒候选。
5. Runtime 仍需检查前序物料步骤和关闭后的稳定保持时间。

## 5. 当前实现

已实现：

- `sop_contracts.v1.Observation`
- 可配置前置、触发转移、后置和保持时间的 `OperationDefinition`
- 顺序 SOP Runtime
- 正确流程、未开盒放物、无手开盒、单帧抖动、视觉不确定和系统中断测试

尚未接入：

- 实际 YOLO 检测输出适配器。
- 目标跟踪器。
- 手部交互关联。
- 相机实时帧。
- 证据视频缓存。

接入这些能力前，不修改 Runtime 的业务判定职责。
