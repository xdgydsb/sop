# 参考视频目标约束

`参考.mp4` 是当前产品形态的首要验收基准。新平台不是旧若依换皮，也不是
固定五步检测页的扩展版。

## 必须形成的产品闭环

1. 数据导入、抽帧、标注、审核和数据版本。
2. 模型训练、评估、发布、版本和回滚。
3. SOP 可视化编排，以及模型、标签、ROI、时序规则绑定。
4. 相机、算力节点、工位、部署任务和健康状态。
5. 真实相机实时检测、步骤判定、异常提示和运行控制。
6. 每一步对应截图、动作视频段、判定依据和任务追溯。

## 复用边界

- 旧若依可复用权限、菜单、基础业务数据和已经验证的运行接口。
- 原检测系统可复用经过真实相机验证的采集、推理和事件能力。
- 所有旧能力必须通过适配器接入，不得决定新平台的信息架构。
- 旧模块若性能差、接口混乱、不可测试或影响目标产品，直接替换。

## 当前阶段验收顺序

| 顺序 | 能力 | 完成条件 |
|---|---|---|
| P0 | 单工位运行闭环 | 真实画面、真实控制、实时步骤、真实证据 |
| P0 | SOP 配置发布 | 可编辑、校验、生成不可变发布版本并部署 |
| P0 | 设备与版本一致性 | 相机、Worker、模型、SOP 版本可见且可恢复 |
| P1 | 数据与模型闭环 | 导入、抽帧、标注、训练、评估、发布 |
| P1 | 多任务追溯 | 周期、步骤、告警、截图和视频段可检索 |

## 禁止偏离

- 不以增加旧 CRUD 页面数量作为平台进度。
- 不用静态演示数据伪装真实运行结果。
- 不因迁就旧接口而删减参考视频中的核心产品流程。
- 不优先做与上述闭环无关的装饰性功能。
- 不在没有回归测试时修改已验证的检测核心。
