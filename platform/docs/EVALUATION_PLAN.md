# 工业工位检测评估计划

## 1. 目标

用真实工位数据回答三个问题：

1. 系统能否及时通过正确动作？
2. 系统能否阻止错误动作被错误通过？
3. 系统故障、遮挡和不确定画面是否会错误归责给工人？

## 2. 数据划分

训练、验证和测试必须按操作员与采集批次划分，不能把同一段视频的相邻帧随机分到不同集合。

建议首轮：

- 至少5名操作员。
- 每人每种正确流程至少10次。
- 每类重点错误至少10次。
- 覆盖快、中、慢三种速度。
- 覆盖轻度遮挡、物体位置变化和合理光照变化。

参考视频和离线录像只能用于算法开发与回归，最终验收必须使用实时相机。

## 3. 首个包装场景测试矩阵

正确测试：

- 标准速度完整执行。
- 较慢执行。
- 较快但动作完整。
- 手短时遮挡盒体。
- 物品放入后盒体标签短时消失。

错误测试：

- 不开盒直接把任意物品放在盒盖上。
- 开盒后跳过指定物品。
- S2/S3/S4错序。
- 物品进入盒内后立即取出。
- 物品只经过盒子上方但没有留存。
- 未完成装料提前关盒。
- 重复放入同类物品。
- 拿错物品。

系统边界测试：

- 相机断开与恢复。
- 推理服务断开与恢复。
- 画面严重模糊或过曝。
- 工人手臂长时间遮挡。
- 页面刷新、任务停止和重置。

## 4. 验收门槛

首轮门槛在获得真实样本后校准，正式演示前至少满足：

- 未开盒放物错误通过率为0。
- 无手参与的静态标签变化不能完成开盒或关盒。
- 单帧误检不能完成步骤。
- 相机或推理中断不能记录为工人失败。
- 每个通过步骤都有对应动作证据。
- 页面、Runtime、数据库和证据状态一致。
- 正常动作判定延迟P95不超过现场可接受阈值。

## 5. 对比实验

每种工序依次比较：

1. YOLO单帧基线。
2. YOLO + 状态稳定。
3. YOLO + 跟踪 + ROI状态转移。
4. YOLO + 跟踪 + 手物交互 + Runtime。
5. 必要时加入在线时序模型。

只有在真实测试指标改善且延迟可接受时，才允许增加更复杂的模型。
