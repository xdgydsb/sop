# 工业作业合规领域模型

## 1. 产品定位

本产品面向工业流水线、装配台、包装台和人工操作工位，通过实时视频判断工人是否按照标准作业程序执行工序，并对漏做、错做、错序、重复、超时和不合格动作进行提示、记录与追溯。

产品的核心不是“画出检测框”，而是回答四个问题：

1. 当前工位正在生产什么、执行哪一版作业标准？
2. 工人刚才是否真实执行了当前要求的动作？
3. 动作的对象、顺序、位置、方向、数量和结果是否正确？
4. 如果错误，什么时候发生、错误原因是什么、证据在哪里？

## 2. 核心业务对象

- `ProductionLine`：产线。
- `Workstation`：工位，绑定相机、算力节点和当前作业。
- `Operator`：操作员，可选工牌、人脸或登录身份。
- `ProductVariant`：产品或规格型号。
- `WorkOrder`：工单，定义生产产品、目标数量和执行范围。
- `SopDefinition`：可编辑的标准作业定义。
- `SopRelease`：不可变的已发布 SOP 版本。
- `OperationDefinition`：一个需要工人完成的动作工序。
- `WorkCycle`：一件产品从开始到结束的实际作业周期。
- `OperationAttempt`：工人在某工序上的一次实际动作尝试。
- `Judgement`：对动作尝试的通过、不通过或无法判断结论。
- `Evidence`：支撑判定的截图、视频段、检测轨迹和规则解释。
- `Deviation`：漏做、错做、错序、重复、超时等偏差事件。

## 3. 工序不是单帧标签

每个 `OperationDefinition` 至少包含：

- 前置条件：动作开始前必须成立的状态。
- 主体条件：是否要求检测到手、工具或特定操作员。
- 操作对象：被拿取、移动、装配或检查的物体。
- 动作语义：拿起、放入、取出、打开、关闭、旋转、按压、连接等。
- 空间约束：来源区域、目标区域、进入方向、包含关系或相对位置。
- 时间约束：连续帧、最短动作时间、完成保持时间和最大允许时间。
- 后置条件：动作结束后必须稳定成立的状态。
- 顺序约束：允许的前序、后序、分支和可选步骤。
- 禁止条件：能够明确判为错误的动作或状态。
- 证据策略：保存动作前、动作中和动作后的画面范围。

因此：

```text
观察到目标标签 != 动作完成

前置状态
  + 人/手/工具参与
  + 对象轨迹或状态转移
  + 空间关系变化
  + 时间稳定性
  + 后置状态
  + SOP 顺序
  = 可解释的工序判定
```

## 4. 判定结果

工序判定不能只有“通过/失败”，至少包括：

- `WAITING`：尚未观察到有效动作。
- `IN_PROGRESS`：动作已经开始但尚未满足完成条件。
- `PASS`：动作和结果均符合当前 SOP。
- `REJECT`：观察到明确错误动作。
- `TIMEOUT`：规定时间内未完成。
- `SKIPPED`：后续动作发生，当前步骤被跳过。
- `UNCERTAIN`：遮挡、画面异常或置信度不足，不能可靠判断。
- `INTERRUPTED`：相机、推理或任务中断，不能归责于工人。

`UNCERTAIN` 和 `INTERRUPTED` 不能伪装成工人操作失败。

## 5. 典型偏差

- 漏做：未执行当前工序就进入后续工序。
- 错序：执行了合法动作，但顺序不符合当前 SOP。
- 错件：拿取或装入了错误物料。
- 错位：物料未进入指定区域或装配位置。
- 数量错误：少装、多装或重复装入。
- 方向错误：零件朝向或插入方向错误。
- 动作不完整：动作发生但未达到稳定后置状态。
- 提前结束：流程未完成就关闭包装或提交成品。
- 超时：单步或整个作业周期超过节拍要求。
- 非标准操作：使用错误工具、越过安全区域或违反动作规则。

## 6. 运行闭环

```text
工单下发
  -> 工位加载 SOP 发布版本
  -> 相机与算力自检
  -> 开始一个 WorkCycle
  -> 实时观察人、物、工具和状态变化
  -> 生成 OperationAttempt
  -> 规则引擎给出 Judgement
  -> 页面即时反馈
  -> 保存 Evidence / Deviation
  -> 完成产品、返工或人工复核
  -> 形成质量和节拍统计
```

## 7. 首个真实场景

五步包装流程可以作为首个场景，但应按工业动作重新表达：

1. 工人在工位内将纸盒从稳定关闭状态打开。
2. 工人将指定耳机从盒外移动到已打开纸盒内部并留存。
3. 工人将指定充电器从盒外移动到纸盒内部并留存。
4. 工人将指定绿色小袋从盒外移动到纸盒内部并留存。
5. 在前三种物料均已正确装入后，工人将纸盒关闭并保持稳定。

静态摆放、手部遮挡、单帧标签变化、把物品放在未打开盒子表面，都不能算作对应动作完成。

## 8. 产品验收原则

- 以真实工位、真实相机、真实操作过程验收。
- 正确动作、错误动作、边界动作和设备故障必须分别测试。
- 判定必须能解释使用了哪些观察、状态转移和规则。
- 页面状态、数据库记录、视频证据和运行引擎结论必须一致。
- 漏报、误报、延迟和无法判断率均需要量化，不能只展示成功案例。
